## Project Overview

Grafana datasource plugin for Litmus Edge that enables real-time data streaming visualization via NATS. TypeScript/React frontend + Go backend.

## Build & Development Commands

- **Frontend build:** `npm run build`
- **Backend build:** `mage build:linux` (Linux amd64 only — do NOT use `mage buildAll`, it cross-compiles all platforms and is much slower)
- **Frontend dev (watch):** `npm run dev`
- **Run Grafana with plugin:** `npm run server` (or `npm run up` for detached)
- **Typecheck:** `npm run typecheck`
- **Lint:** `npm run lint` (fix: `npm run lint:fix`)
- **Unit tests:** `npm run test:ci`
- **Backend tests:** `mage test`
- **E2E tests:** `npm run e2e` (first run: `npm run e2e:install`)
- **Stop Grafana:** `npm run down`

## Important Codebase Rules

- **Never edit files in `.config/`** — this directory is auto-generated by Grafana Plugin Tools. Extend configs from root-level files instead (e.g., `tsconfig.json`, `eslint.config.mjs`, `.prettierrc.js`).
- The Go backend binary name is `gpx_edge` — must match the `executable` field in `src/plugin.json`.
- Frontend uses Grafana Live (WebSocket streaming), not HTTP requests.

## Environment Setup

- Node.js 22+ (see `.nvmrc`), Go version from `go.mod`, Mage, Docker with Compose v2
- Copy `.env.example` to `.env` and set `LITMUS_EDGE_HOSTNAME` and `LITMUS_EDGE_TOKEN`

## Commit Style

Use conventional commits: `feat:`, `fix:`, `chore:`, `refactor:`, `ci:`, `docs:`, `test:`
