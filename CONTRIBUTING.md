# Contributing

How to set up the project, build it, and run tests.

## Requirements

- Node.js 22+ (see `.nvmrc`)
- Go (see version in `go.mod`)
- Docker with Compose v2
- [Mage](https://magefile.org/): `go install github.com/magefile/mage@latest`

The project uses [Grafana Plugin Tools](https://grafana.com/developers/plugin-tools).

## Setup

```bash
npm install

cp .env.example .env
# For inside-LE mode (default), set EDGE_DOCKER_GATEWAY_IP if this instance
# does not use the default gateway IP of 10.30.50.1.
# EDGE_API_TOKEN is optional but recommended for topic discovery.
# For external Litmus Edge: set EDGE_HOSTNAME (Litmus Edge address),
# EDGE_ACCESS_ACCOUNT_TOKEN, and EDGE_EXTERNAL=true.
# Set EDGE_NATS_PROXY_PORT if the live stream does not use 4222.
```

## Build

```bash
# Full plugin build (frontend + backend)
npm run build

# Frontend only (outputs to dist/)
npm run build:frontend

# Backend only, Linux amd64 (outputs binary to dist/). This script also resolves `mage` from common Go bin locations.
npm run build:backend
```

Don't use `mage buildAll`. It cross-compiles for every platform and takes much longer.

## Run Grafana locally

```bash
npm run server    # foreground, logs visible
npm run up        # detached
npm run down      # stop
```

The dev Docker image mounts `dist/` and `provisioning/` into Grafana. Set `GRAFANA_IMAGE=grafana-oss` if you want OSS instead of Enterprise.

## Development workflow

1. Start Grafana: `npm run server`
2. In a second terminal: `npm run dev` (watches and rebuilds the frontend)
3. Edit code. The frontend rebuilds automatically; refresh the browser to see changes.
4. Backend changes need a rebuild (`mage build:linux`) and a restart (`npm run restart`).

## Checks before a PR

```bash
# Frontend
npm run typecheck    # TypeScript
npm run lint         # ESLint (fix: npm run lint:fix)
npm run test:ci      # Jest

# Backend
mage test            # Go tests
```

## Plugin validation

Use Grafana's Dockerized plugin validator against the current build output:

```bash
npm run validate:plugin
```

To validate a release archive instead:

```bash
npm run validate:plugin -- https://github.com/litmusautomation/edge-datasource/releases/download/v0.1.0/litmus-edge-datasource-0.1.0.zip
```

Optional environment variables:

- `VALIDATOR_FLAGS` — extra validator flags such as `-strict`
- `VALIDATOR_SOURCE_CODE_URI` — override the source code URI passed to the validator

The first run can take a minute while Docker layers and validator analyzers initialize.

## E2E tests

These need a running Grafana with a reachable Litmus Edge device configured in `.env`.

```bash
npm run e2e:install  # first time only, installs Chromium
npm run e2e          # run tests
npm run e2e:ui       # interactive mode
```

## Code organization

```
src/                  TypeScript/React frontend
  components/         Query and config editor components
  datasource.ts       DataSource class (streaming via Grafana Live)
  types.ts            Shared types
  plugin.json         Plugin metadata
pkg/                  Go backend
  edge/               NATS client, topic management, message framing
  plugin/             Grafana SDK integration (health, streaming, query)
provisioning/         Grafana provisioning files for local dev
.config/              Auto-generated by Grafana Plugin Tools (don't edit)
```
